# Image Naming Guide for Banking History Site

This guide provides the exact naming conventions for all historical images you source. All images should be placed in the `/public/images/` directory with chapter-specific subdirectories.

## Directory Structure

```
/public/images/
  ├── federal-reserve/
  ├── biblical-prophecy/
  ├── modern-control/
  ├── fiat-system/
  ├── industrial-revolution/
  ├── banking-houses/
  ├── medieval-banking/
  └── ancient-foundations/
```

## Image Format Guidelines

- **Format**: JPG or PNG (JPG preferred for photographs, PNG for diagrams)
- **Resolution**: Minimum 1200px width for good quality on modern displays
- **File Size**: Optimize to keep under 500KB when possible
- **Naming**: Use lowercase with hyphens, descriptive names

## General Guidelines

### Finding Public Domain Images

**Recommended Sources:**
- **Library of Congress** (loc.gov) - Vast historical photo archive
- **National Archives** (archives.gov) - Government historical records
- **Wikimedia Commons** (commons.wikimedia.org) - Public domain images
- **Public Domain Review** (publicdomainreview.org) - Curated historical images
- **Met Museum** (metmuseum.org) - Art and historical artifacts
- **Europeana** (europeana.eu) - European cultural heritage
- **New York Public Library Digital Collections** (digitalcollections.nypl.org)

### Image Quality Checklist

✅ High resolution (minimum 1200px width)  
✅ Clear, not grainy or pixelated  
✅ Public domain or properly licensed  
✅ Relevant to content  
✅ Named exactly as specified in this guide  
✅ Placed in correct subdirectory  
✅ Optimized file size (under 500KB when possible)  

### After Sourcing Images

Once you've downloaded and renamed images according to this guide:

1. Create the directory structure in `/public/images/`
2. Place each image in its corresponding chapter subdirectory
3. Ensure filenames match exactly (case-sensitive)
4. Verify images display correctly in development server
5. Check that alt text and captions are appropriate

The code has been updated to automatically display these images in the correct sections once they're added to the `/public/images/` directory.
